# 단어시험 관리 프로그램

단어 시험 점수를 입력하면 자동으로 통과/재시험 여부를 판정하고, 재시험 일정 예약과
학생·반·회독별 통계를 관리하는 웹 프로그램입니다.

---

## ▶ 실행 방법 (가장 쉬움)

1. 폴더 안의 **`시작하기.bat`** 파일을 더블클릭하세요.
2. 검은 창이 뜨고, 잠시 후 브라우저에 프로그램이 자동으로 열립니다.
   - 주소: `http://localhost:3000`
3. **끄려면** 그 검은 창을 닫으면 됩니다.

> 최초 1회는 구성요소 설치 때문에 시간이 조금 걸립니다.

> 실행은 **프로덕션 모드(`npm start`)** 로 동작해 첫 화면부터 즉시·완전하게 작동합니다.
> 검은 창은 서버가 준비될 때까지 기다렸다가 브라우저를 자동으로 엽니다.

### 명령어로 실행하려면
```powershell
npm install   # 최초 1회
npm run build # 코드가 바뀌면 다시 실행
npm start     # 서버 시작 → http://localhost:3000
```

> ⚠️ 소스 코드를 수정한 뒤에는 `npm run build` 를 다시 한 번 실행해야 변경이 반영됩니다.
> (개발 중 즉시 반영이 필요하면 `npm run dev` 사용)

## 로그인 (선생님 / 학생)

프로그램을 처음 열면 **로그인 화면**이 나옵니다.

- **선생님**: 처음 한 번 비밀번호를 정하면 그 비밀번호로 로그인합니다. (전체 관리 화면)
- **학생**: 선생님이 학생을 추가하면 계정이 자동 발급됩니다.
  - **아이디 = 학생 이름**, **초기 비밀번호 = 0000** (모두 동일). 이름이 겹치면 뒤에 숫자가 붙습니다(예: 김민준2).
  - 학생에게 "이름으로 로그인, 비번 0000"이라고 안내하고, **로그인 후 비밀번호를 바꾸도록** 하세요. (분실 시 "비번 발급"으로 0000 재설정)
  - 학생은 로그인 후 **본인 성적·재시험 일정·통과 여부만** 볼 수 있고, **본인 재시험 예약**을 할 수 있습니다. (점수 입력은 선생님)
  - 학생은 로그인 후 비밀번호를 바꿀 수 있습니다.

> 🔒 학생은 서버에서 본인 데이터만 받도록 막혀 있어, 다른 학생 성적은 볼 수 없습니다.

## 사용 순서 (선생님)

1. **관리(⚙️) 탭**
   - 반 만들기 (요일: 월수금 / 화목, 통과 컷 % 설정)
   - 학생 추가 (쉼표·줄바꿈으로 여러 명 한 번에)
   - 학생 퇴원 처리/복원 (점수 기록은 보존, 새 점수 입력·통계 재원생 목록에서는 제외)
   - 책/단어장 추가 (기본 만점, 책별 컷 — 비우면 반 컷 사용)
2. **점수 입력(✏️) 탭**
   - 반·이름·책·만점·실제성적·회독 입력 → **자동으로 통과/재시험 판정**
   - 사진(시험지) 첨부 가능
   - 미통과 시 곧바로 **재시험 일정 예약**(10분 단위)
3. **재시험(🔁) 탭**
   - 예약된 재시험 목록과 남은 시간 확인
   - 재시험 결과 입력 → 자동 재판정 → 통과면 종료, 미통과면 **재재시험 예약**
4. **통계(📊) 탭**
   - 반/학생/회독(1~3)별 평균과 추이 그래프
   - 선생님용 표: 학생별 **만점 횟수·통과 횟수·만점 연속·통과 연속**
   - 기본 통계 기간은 **여름학기 성취(2026.06.10 이후)**입니다.
   - 여름학기 성취 통계는 **정규 시험만** 계산하고, 기준은 **11회 시험 중 9회 이상 통과**입니다. 재시험은 성취 횟수에서 제외됩니다.
   - 이전 데이터는 **봄학기(6/5까지)** 필터로 따로 확인할 수 있습니다.

---

## 통과 판정 기준

두 가지 컷 방식을 지원합니다 (책별로 선택):
- **절대 점수 컷** (권장, 예: "51점 이상 통과") — 관리 탭에서 책의 "통과 점수(컷)"에 점수를 입력. 그 점수 이상이면 통과.
- **백분율 컷** — "컷(%)"에 입력. (실제 ÷ 만점 × 100) ≥ 컷% 이면 통과.
- 절대 점수 컷을 넣으면 그게 우선합니다. 컷을 바꾸면 해당 책의 기록이 **자동 재판정**됩니다.
- 점수는 **소수점(.5)** 까지 입력할 수 있습니다.

## 데이터 저장 위치

- 모든 데이터: `data/db.json` (이 한 파일에 전부 저장)
- 첨부 사진: `data/uploads/`
- 백업하려면 `data` 폴더를 통째로 복사하세요.

## ⚠️ 알림(푸시)에 대한 안내 — 2단계

요청하신 **재시험 24시간 전·2시간 전 휴대폰 푸시 알림**은 다음이 필요합니다.

- 인터넷 HTTPS 호스팅 (학생들이 각자 폰으로 접속)
- 항상 켜져 있는 서버 (예약 시각에 알림 발송)

현재(1단계)는 선생님 PC에서 도는 로컬 버전이라, 재시험 현황 화면에서
**일정과 남은 시간**을 확인하는 형태입니다. 푸시 알림은 무료 클라우드 배포(2단계)
때 추가됩니다.

## 기술 구성

- Next.js 15 + React 19 + TypeScript + Tailwind CSS
- 데이터: JSON 파일 저장소 (별도 DB 서버 불필요)
- 그래프: Recharts

## 폴더 구조

```
src/
  app/
    page.tsx              메인(탭 화면)
    api/state/            현재 데이터 조회
    api/command/          데이터 변경(액션)
    api/upload/           사진 업로드
  components/             화면별 컴포넌트
  lib/
    types.ts              데이터 구조 정의
    actions.ts            서버 액션(통과 판정·재시험 로직)
    logic.ts              판정·통계·연속횟수 계산
    db.ts                 JSON 저장소
    datetime.ts           날짜·10분 단위 시간
data/
  db.json                 전체 데이터
  uploads/                첨부 사진
```
